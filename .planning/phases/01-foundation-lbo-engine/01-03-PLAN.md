  <action>
    RULE: No financial math in any component. No parsing in any component. Components read from stores and call store actions only.

    Create `src/hooks/useActiveOutputs.ts`:
    ```typescript
    import { useModelStore } from '../stores/modelStore';
    export function useActiveOutputs() {
      const outputs = useModelStore((s) => s.outputs);
      const activeScenario = useModelStore((s) => s.activeScenario);
      return outputs[activeScenario];
    }
    ```

    Create `src/components/scenario/ScenarioToggle.tsx`:
    - Three buttons: Base / Bull / Bear
    - Active scenario highlighted (use Tailwind: bg-blue-600 text-white for active, bg-gray-100 for inactive)
    - On click: `useModelStore((s) => s.setActiveScenario)(scenario)`
    - No computation triggered — purely calls setActiveScenario

    Create `src/components/upload/UploadZone.tsx`:
    - Accept: '.csv,.pdf'
    - Drag-and-drop zone with dotted border (Tailwind: border-2 border-dashed border-gray-300 rounded-lg p-8)
    - On file selected: check file.type or file.name extension
    - CSV path: call `parseCSV(file)`, on success call `dataStore.setFinancials(result)`, show success message
    - PDF path: call `extractPDFText(file)`, on success call `dataStore.setFinancials(parsedFromText)` where parsedFromText is a stub ParsedFinancials with source: 'pdf' and rawText
    - On error (including scanned PDF error): show the error.message in a red alert div
    - Show loading state while parsing: "Parsing file..."
    - State: local React state for loading/error — these are UI states, not store states

    Create `src/components/upload/ParsedPreview.tsx`:
    - Reads parsedFinancials from dataStore
    - Renders a table with rows: Revenue, EBITDA, Capex, D&A; columns: one per year
    - Each cell is an `<input type="number">` pre-populated with the parsed value (formatted as plain number, no $MM here — user edits raw)
    - On input change: call `dataStore.overrideField(field, yearIndex, event.target.value)`
    - Show confidence indicator: if confidence[field] === 'low', show a yellow dot next to the label
    - Show parseWarnings as a dismissable alert above the table
    - If parsedFinancials is null: show nothing (UploadZone shows instead)

    Create `src/components/lbo/AssumptionPanel.tsx`:
    - Props: `scenario: ScenarioKey`
    - Reads `useModelStore((s) => s.scenarios[scenario])` for current values
    - Renders inputs for: Entry EBITDA ($M), Entry Multiple (x), Equity % (%), Management Rollover ($M), Exit Multiple (x), Exit Year
    - For debt tranches: render per-tranche sub-panel with Principal ($M), Rate (%), Mandatory Amort (%)
    - On change: call `modelStore.setAssumption(scenario, fieldName, newValue)` — use debounced input (300ms) via a local setTimeout ref to avoid calling computeLBO on every keystroke
    - Constraint reminder: show static text "Sponsor equity = Total equity - Rollover (enforced)" near the rollover input

    Create `src/components/lbo/SourcesUses.tsx`:
    - Reads outputs from `useActiveOutputs()`
    - If outputs is null: show "Configure assumptions to see Sources & Uses"
    - Renders a two-column table: Sources (Sponsor Equity, Management Rollover, TLA, TLB) and Uses (Purchase Price)
    - All values through `formatFinancial(value, 'currency_mm')`
    - Leveraged entry = totalDebt / entryEV — show as percentage_2dp

    Create `src/components/lbo/DebtScheduleTable.tsx`:
    - Reads `useActiveOutputs()?.debtSchedule`
    - If null: show a dash
    - Renders table: Year | Tranche | Beginning Balance | Interest | Mandatory Amort | Cash Sweep | Ending Balance
    - All dollar values through `formatFinancial(value, 'currency_mm')`
    - Interest rate column: `formatFinancial(rate, 'percentage_2dp')`
    - Show converged indicator: if !outputs.converged, show yellow warning "Debt schedule did not fully converge"

    Create `src/components/lbo/OutputTable.tsx`:
    - Reads all three scenario outputs from `useModelStore((s) => s.outputs)`
    - Renders a comparison table: Row = metric, Column = Base / Bull / Bear
    - Metrics: IRR (percentage_2dp), MOIC (multiple_2dp), Entry EV (currency_mm), Exit EV (currency_mm), Exit Equity (currency_mm), Sponsor Equity (currency_mm)
    - If any scenario output is null: show a dash in that column
    - Highlight the active scenario column with a light blue background

    Update `src/App.tsx`:
    - Layout: full-page, dark professional background (Tailwind: bg-gray-950 text-gray-100)
    - Header: "PE Deal Workbench" in a top bar
    - Main content: two-column layout on lg screens (lg:grid lg:grid-cols-3)
    - Left column (col-span-1): UploadZone, ParsedPreview, ScenarioToggle, AssumptionPanel (for activeScenario)
    - Right column (col-span-2): SourcesUses, OutputTable, DebtScheduleTable
    - All components imported and rendered
    - Do NOT add shadcn/ui components in this plan — use plain Tailwind for all styling. shadcn/ui can be added in Phase 2 for polish.
