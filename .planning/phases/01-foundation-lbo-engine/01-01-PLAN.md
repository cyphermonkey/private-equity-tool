    Implement XIRR using Newton-Raphson. The implementation in 01-RESEARCH.md (Pattern 3) is the reference — implement it exactly:
    - Type: `export interface CashflowPair { amount: number; date: Date; }`
    - `export function xirr(cashflows: CashflowPair[], guess = 0.1): number`
    - Use t0 = cashflows[0].date as the base date
    - NPV: sum of `cf.amount / (1 + rate)^t` where t = days from t0 / 365
    - dNPV: derivative for Newton-Raphson
    - Convergence: |newRate - rate| < 1e-7 within 100 iterations
    - Throw: `new Error('XIRR did not converge')` if not converged
    - NOTE: xirr operates on native JS number (not Decimal) — the outer computeLBO will convert

    Create `src/engine/lbo.ts`:

    Implement `export function computeLBO(a: LBOAssumptions): LBOOutputs`:

    Step 1 — Sources & Uses:
    ```
    entryEV = new Decimal(a.entryEBITDA).times(a.entryMultiple)
    totalEquity = entryEV.times(new Decimal(a.equityPct).dividedBy(100))
    sponsorEquity = totalEquity.minus(a.managementRollover)   // HARD CONSTRAINT
    totalDebt = entryEV.minus(totalEquity)
    ```

    Step 2 — EBITDA/Revenue projection: for each year 1..exitYear, compute:
    ```
    revenue[0] = prior year revenue * (1 + growthRate/100)
    ebitda[y] = revenue[y] * (margin/100)
    // Use a baseRevenue derived from entryEBITDA / entryEBITDAMargin for year 0
    // Treat entryEBITDA as year 0 EBITDA; revenue year 0 = entryEBITDA / (margin[0]/100)
    ```

    Step 3 — Debt schedule with fixed-point iteration per tranche per year:
    - For each tranche, for each year 1..exitYear:
      - mandatoryAmort = Decimal(tranche.principal).times(tranche.mandatoryAmortPct).dividedBy(100)
      - Run fixed-point iteration (max 10 iterations, convergence delta < 0.01):
        - interestExpense = currentBalance.times(tranche.rate).dividedBy(100)
        - EBITDA available for sweep (net of all tranche interests + amort, simplified to per-tranche basis for convergence)
        - cashSweep = max(0, ebitda[year] - interestExpense - mandatoryAmort) but capped at remaining balance
        - newBalance = max(0, currentBalance - mandatoryAmort - cashSweep)
        - If |newBalance - currentBalance| < 0.01 → break
        - Update currentBalance = newBalance
      - Record DebtYearRow
    - Set converged = true if all iterations converged within 5 passes

    Step 4 — Exit:
    ```
    exitEBITDA = projected ebitda at exitYear
    exitEV = new Decimal(a.exitMultiple).times(exitEBITDA)
    totalEndingDebt = sum of all tranche ending balances at exitYear
    exitEquity = exitEV.minus(totalEndingDebt)
    ```

    Step 5 — XIRR / MOIC:
    - Build cashflowPairs: entry = [{amount: -sponsorEquity.toNumber(), date: closingDate}], exit = [{amount: exitEquity.toNumber(), date: addYears(closingDate, exitYear)}]
    - irrValue = xirr(cashflowPairs)
    - moicValue = exitEquity.dividedBy(sponsorEquity)

    Return LBOOutputs with all values as Decimal.toString() strings. Do NOT call formatFinancial() inside computeLBO — formatting is a UI concern, not an engine concern.

    Pitfall guard — Sources & Uses: verify that sponsorEquity is positive (throw if managementRollover >= totalEquity).
    Pitfall guard — Debt schedule: throw `new Error('Debt schedule did not converge for tranche ${name}')` if delta > 1 after 10 iterations.
    Pitfall guard — XIRR: wrap xirr() call in try/catch; if it throws, set irr to 'NaN' and converged to false, do not crash computeLBO.
